# Javideo

本地番号影片刮削、磁力搜索与媒体库管理工具（Windows 桌面软件）。

![search](https://img.shields.io/badge/status-v1.0.0-green)
![platform](https://img.shields.io/badge/platform-Windows-blue)

---

## 功能一览

### 搜索与刮削

输入番号，自动从 MetaTube 刮削元数据（封面、标题、演员、简介、厂商、发行日期等），
搜索磁力链接，并获取 DMM 预告片（需代理）。刮削结果支持一键入库。

- 元数据翻译（接入 AI 翻译服务可自动翻译标题与简介）
- 多候选源选择（同一番号有多个数据源时供用户挑选）

### 媒体库管理

入库后的影片按媒体库组织，文件夹结构为：

```
{媒体库目录}/
  {番号}/
    poster.jpg         封面
    thumb.jpg          缩略图
    {番号}.nfo         Kodi 兼容的 nfo 元数据
    磁力链接.txt        关联的磁力链接
```

- 支持多个媒体库
- 扫描目录识别已下载的影片文件（mp4/mkv/strm）
- 批量入库扫描结果

### 收藏

三类收藏入口，集中管理常看内容：

- **影片收藏** — 按最近收藏或番号 ID 排序，支持批量取消
- **标签收藏** — 收藏常用标签快速筛选
- **演员收藏** — 收藏关注的演员

### 演员

浏览演员信息与作品列表，支持分页。演员详情页展示生日、身高、三围、罩杯、血型等基本信息。

### 标签

按以下维度浏览：

- **类型（Genre）**
- **系列（Series）**
- **厂商（Maker）**
- **自定义标签**

支持标准库与非标准库的标签聚合。

### 播放器

- 自定义播放器路径，未配置时使用系统默认播放器
- 入库影片可直接从详情页启动播放

### 设置

| 模块 | 功能 |
|------|------|
| 媒体库 | 新建/编辑/删除媒体库，管理扫描目录 |
| MetaTube | 配置刮削服务地址、超时、连通性测试 |
| 基本设置 | 语言（中/英）、主题（深色/浅色）、关闭行为、调试模式 |
| 播放器 | 自定义播放器路径 |
| 网络代理 | 配置 HTTP 代理用于下载 DMM 预告片 |
| 数据导入导出 | 备份与恢复用户数据 |

## 快速开始

1. 前往 [Releases](https://github.com/CowanNath/Javideo/releases) 下载最新版
2. 解压后保持 `javideo.exe` 与 `javideo-worker.exe` 在同一目录
3. 双击 `javideo.exe` 启动
4. 首次使用前往「设置 → MetaTube」配置刮削服务地址
5. 在「设置 → 媒体库」新建媒体库
6. 回到搜索页搜索番号，刮削后入库即可

> 提示：若需刮削 DMM 预告片，请在「设置 → 网络代理」配置可用代理。

## 架构

```
┌─────────────────────────────────────────────────────┐
│                    javideo.exe                       │
│  ┌──────────────────────────────────────────────┐   │
│  │         Tauri 壳 (Rust)                      │   │
│  │  - 启动 / 管理 worker 进程                    │   │
│  │  - 暴露 get_worker_url 给前端                │   │
│  └──────────────┬───────────────────────────────┘   │
│  ┌──────────────┴───────────────────────────────┐   │
│  │         Vue 3 前端                            │   │
│  │  - 搜索/收藏/演员/标签/设置                   │   │
│  │  - Pinia 状态管理 / UnoCSS 样式               │   │
│  └──────────────┬───────────────────────────────┘   │
│                 │ fetch (127.0.0.1:动态端口)         │
│  ┌──────────────┴───────────────────────────────┐   │
│  │    javideo-worker.exe (.NET 8 Kestrel)       │   │
│  │  - MetaTube 刮削客户端                       │   │
│  │  - 多源磁力搜索（Pollack / Btdig / Yhg007）  │   │
│  │  - SQLite 数据持久化                         │   │
│  │  - NFO / 封面 / 缩略图写入                   │   │
│  │  - DMM 预告片下载                            │   │
│  │  - AI 翻译服务                               │   │
│  └──────────────────────────────────────────────┘   │
└─────────────────────────────────────────────────────┘
```

Worker 以 Tauri **sidecar** 方式启动，监听 `127.0.0.1:0`（系统分配端口），
启动后在 stdout 打印 `JAVIDEO_WORKER_PORT=12345`，Tauri 壳读取后暴露给前端。

## 数据来源

- **元数据刮削**：[MetaTube](https://github.com/metatube-community/metatube-sdk-go)（用户自填服务器地址）
- **磁力搜索**：pollack3.sbs（HTML 解析），预留 Btdig / Yhg007 接口
- **预告片**：DMM 官方（需非日本 IP 或代理）
- **翻译**：接入 LLM API（需在设置中配置）

## 合规说明

- 本软件为本地媒体库管理工具，**不内置任何下载器**
- 磁力链接仅以文本形式展示并写入 txt，由用户自行处理
- 不涉及下载、分发或版权内容的托管
